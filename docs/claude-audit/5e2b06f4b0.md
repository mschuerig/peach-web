
However, that would break `cargo test -p domain` (which runs natively). The better approach is to leave the default target as native and accept that RustRover may flag some `web-sys`/`wasm-bindgen` imports with warnings. `cargo clippy -p web --target wasm32-unknown-unknown` remains the authoritative check.

**Tailwind classes** — The utility classes in Rust string literals won't get IntelliSense. That's just the nature of Tailwind-in-Rust — no IDE currently handles this well.

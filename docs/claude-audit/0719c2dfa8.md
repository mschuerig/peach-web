### CRITICAL / HIGH ISSUES

**H1 — AC10 violation: `stop()` doesn't reset all session-level state** `domain/src/session/comparison_session.rs:242-250`

AC10 requires: "state returns to idle, current comparison is discarded, and **all session-level state resets**". Task 5.1 also specifies: "clear session-level transient state".

`stop()` clears `current_comparison`, `current_playback_data`, and `show_feedback` — but leaves these stale:
- `is_last_answer_correct` — observable via accessor, will show the last session's answer result
- `session_best_cent_difference` — observable via accessor, will show last session's best
- `last_completed` — internal, used by `generate_next_comparison()`

While `start()` resets these on the next session, the AC requires `stop()` to do it. Between `stop()` and the next `start()`, the UI could display stale data (e.g., showing "correct" from a previous session).

---

### MEDIUM ISSUES

**M1 — `FEEDBACK_DURATION_SECS` not re-exported** `domain/src/session/mod.rs:3` and `domain/src/lib.rs:15`

The constant is `pub` in `comparison_session.rs` but missing from re-exports in both `mod.rs` and `lib.rs`. The web layer (story 1.7) will need this constant and would have to use the verbose path `domain::session::comparison_session::FEEDBACK_DURATION_SECS` instead of `domain::FEEDBACK_DURATION_SECS`.

**M2 — `current_interval()` silently swallows errors** `domain/src/session/comparison_session.rs:114-122`

```rust
DirectedInterval::between(c.reference_note(), c.target_note().note)
    .unwrap_or(DirectedInterval::new(
        crate::types::Interval::Prime,
        crate::types::Direction::Up,
    ))
```

If `DirectedInterval::between()` fails (e.g., compound interval > octave), the error is silently replaced with a `Prime/Up` fallback. This violates project-context.md: "DO NOT swallow errors". Should at least log via `eprintln!` when the `Result` is `Err`, even if the fallback is kept for UI resilience.

## Commit message

Fix is_target_higher incorrectly ignoring interval for non-unison comparisons

is_target_higher() only checked the detuning offset sign, which is wrong
for non-unison intervals (e.g. a major third up with -50 cent offset is
still higher overall). Now accounts for the MIDI note difference converted
to cents plus the offset.

Also adds a training-log Cargo feature flag that logs each comparison's
details and answer result to the browser console, and logs user button
presses (including ignored ones) for debugging input registration issues.

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
